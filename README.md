# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 일상 생활 정보 기반 건강 상태 확인 프로그램

<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 사용자의 키와 체중을 이용하여 BMI를 계산하고, 추가적으로 성별 및 하루 평균 운동시간을 입력받아 예상 기초대사량(BMR)과 총 소비 칼로리(TDEE)를 계산하여 건강 상태를 분석하고 관리하는 개인 맞춤형 건강 및 에너지 관리 시스템입니다.
* 사용자는 자신의 키, 몸무게, 성별, 하루 운동 시간을 입력합니다. 시스템은 이를 바탕으로 체질량지수(BMI)를 계산하고, 특히 성별에 따른 서로 다른 수식을 적용하여 기초대사량에 따른 하루 총 예상 소비 칼로리(TDEE)를 산출합니다. 또한, 계산된 BMI 수치를 기준으로 건강 상태 등급(저체중, 정상, 과체중, 비만, 고도비만)을 판별하며, 특정 조건(예: 정상 체중이면서 하루 운동 시간이 1시간 이상인 경우)을 만족하면 '갓생 사는 건강왕'과 같은 특별 칭호를 부여하는 로직이 포함됩니다. 최종적으로는 메뉴판을 통해 원하는 분석 기능을 선택하고, 본인의 건강 데이터를 정밀하게 추적 및 관리할 수 있는 시스템을 완성할 계획입니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
 
1. 기본 정보 입력
2. 기초 건강 지수 계산 
3. 에너지 소비 데이터 분석
4. 상태 판별 및 칭호 부여
5. 개인 건강 리포트 출력
0. 시스템 종료

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * 사용자로부터 성별, 키, 몸무게, 하루 평균 운동시간을 입력받아 BMI(체질량지수)와 총 예상 칼로리(TDEE)를 계산하여 출력합니다.
  * 사용자의 성별(문자), 키/몸무게(정수), 운동시간(실수) 데이터를 입력받는 화면을 구현했습니다. 
  * 사칙연산을 활용해 신체 지수와 활동량이 반영된 BMI 및 하루 총 예상 소비 칼로리를 계산하여 출력했습니다. (성별에 따른 TDEE계산은 다음 버전 때 적용)
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "인바디같은 기계 없이도 편리하게 건강을 확인하는 시스템을 목표로, C언어 4단계 과제 로드맵에 적합한 시나리오 및 기능 확장 로드맵 기획 요청"
    * **적용 내용:** '기본 정보 입력(1차) ➔ 성별 맞춤 분석 및 등급 판별(2차) ➔ 메인 메뉴 시스템(3차) ➔ 배열/함수 기반 데이터 관리(4차)'로 이어지는 체계적인 확장 안을 수립함. 단순 수치 출력을 넘어 '목표 설정'이 가능한 시스템으로 구현함.
      
 2. **내용 2**
    * **프롬프트 요약:** "1차 과제 필수 요건(변수 5개, 자료형 3개 이상)을 충족하기 위한 초기 기본 정보 구조 모델링 논의 및 효율적 건강 지표 계산법 질문"              
    * **적용 내용:**  데이터 특성에 맞춰 문자형(char - 성별), 정수형(int - 키, 몸무게), **실수형(float - 운동 시간)**으로 자료형을 명확히 분리하여 입력 데이터의 구조적 완성도를 높임. 특히 키와 몸무게를 정수형으로 설정하여 입력의 가독성을 높였으며, 운동 시간은 시간 개념(예: 1.5시간)에 맞게 실수형으로 처리하여 BMI 및 TDEE 계산 시 정밀한 분석이 가능하도록 설계함.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 출력 중 오류가 있어 결과를 볼 수 없음.
     * **해결:** AI에게 질문하여 ;을 생략한 것을 확인 후, 코드를 수정하여 해결함.
  2. **문제 2:** 실수형 데이터를 출력할 때 소수점이 너무 길게 나옴.
     * **해결:** 구글링과 AI 검색을 통해 `printf("%.2f", weapon_power);`와 같이 서식 지정자를 수정하여 소수점 둘째 자리까지만 출력되도록 개선함.
